File API Reference

All endpoints require Authorization: Token <key> header.

Authentication
POST/api/auth/login/
Body: {"username":"…","password":"…"} → returns token
POST/api/auth/logout/
Ends the session.
Files
GET/api/v1/files/
List all files and folders accessible to the authenticated user.
POST/api/v1/files/upload_multiple/
Upload files. Form-data: files[], optional parent (folder id).
GET/api/v1/files/<id>/download/
Download a file.
POST/api/v1/files/<id>/trash/
Move a file to trash.
PATCH/api/v1/files/<id>/
Rename or move a file. Body: {"name":"…"} or {"parent":"<folder_id>"}
Sharing
POST/api/v1/permissions/
Share a file. Body: {"file":"<id>","grantee_email":"…","role":"reader|writer|commenter"}
DELETE/api/v1/permissions/<id>/
Revoke a share permission.
Trash
GET/api/v1/trash/
List trashed files.
POST/api/v1/trash/<id>/restore/
Restore a file from trash.
DELETE/api/v1/trash/<id>/purge/
Permanently delete a file.
Example request
curl -X POST /api/v1/permissions/ \
  -H "Authorization: Token <your_token>" \
  -H "Content-Type: application/json" \
  -d '{"file":"<file_id>","grantee_email":"user@example.com","role":"reader"}'
